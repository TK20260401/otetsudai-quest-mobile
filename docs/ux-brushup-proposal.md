- **明示的ルール**: 「クエストをクリア→おやがOK→おかねがもらえる」のフロー図を常時表示

- **セマンティックHTML的RN構造**: accessibilityRole="header", "button", "adjustable"
- **フォーカス管理**: 画面遷移時にuseRef + AccessibilityInfo.setFocusTo
- **ライブリージョン**: accessibilityLiveRegion="polite" for dynamic updates
- **触覚フィードバック**: expo-haptics でタップ/成功/エラーを振動パターンで区別
- **Magic Tap**: accessibilityActions for quick quest completion

## 2. 挫折ポイント別 具体的UI/UX改善案

### 挫折ポイント1: 操作の困難さ（特に特性のある子）

| 案 | 具体案 | コンポーネント/マイクロインタラクション | 文言例 | 優先度 | ROI |
|---|-------|--------------------------------------|--------|-------|-----|
| 1-A | **タッチターゲット拡大＋フィードバック強化** | `clearButton`: paddingVertical 10→16, min height 56pt。`Animated.spring`で0.95→1.0倍スケール。expo-haptics `impactAsync(Medium)` | 「クリア！」 | **High** | 全ユーザーの操作ミス率低減 |
| 1-B | **段階的UI開示（Progressive Disclosure）** | 初回ログイン時にウォレット・バッジを非表示、クエストのみ表示。2回目以降に段階開放。`AsyncStorage`で`onboarding_step`管理 | 「まずは クエストを やってみよう！」 | **High** | ASD/知的障害の情報過多ストレス軽減 |
| 1-C | **操作確認ダイアログ** | クエストクリア時に`AppAlert`で確認。キャンセル可能。予測可能性向上 | 「『さらあらい』を クリアする？」→「はい」/「やめる」 | **Mid** | ASD児の不安軽減。定型発達児には煩わしい可能性→設定でON/OFF |

**実装観点**: 1-Aは既存StyleSheetの数値変更＋Animated.springの追加のみ。expo-hapticsは`npx expo install expo-haptics`で導入。1-Bは`AsyncStorage`に状態保存、条件分岐でセクション表示制御。1-Cは既存`AppAlert`コンポーネントをそのまま利用可能。

---

### 挫折ポイント2: 初期報酬の低さ（開始直後の離脱）

| 案 | 具体案 | コンポーネント/マイクロインタラクション | 文言例 | 優先度 | ROI |
|---|-------|--------------------------------------|--------|-------|-----|
| 2-A | **ウェルカムボーナス** | 初回ログイン時に100円を自動付与。`LevelUpModal`類似の演出で「スタートボーナス！」表示。コインが降ってくるアニメーション | 「ぼうけんのスタートボーナス！ 100えん もらったよ！」 | **High** | 初回の空ウォレット問題を即座に解消。行動経済学の「保有効果」活用 |
| 2-B | **初回3クエストのダブル報酬** | `otetsudai_task_logs`のカウントで判定。承認時にreward × 2を付与。「ビギナーボーナス ×2」バッジ表示 | 「🌟 ビギナーボーナス！ いまなら 2ばい！（あと2かい）」 | **Mid** | Duolingo等で実証済みの初期ブースト。3回で打ち切りにより過剰報酬を回避 |
| 2-C | **マイルストーン演出の前倒し** | 現在の最低バッジ「はじめてのクエスト」を演出強化。初回承認時に`LevelUpModal`並みの全画面演出＋効果音 | 「🎉 はじめてのクエスト クリア！ きみは もう ぼうけんしゃだ！」 | **High** | コスト最小（既存コンポーネント流用）で初回体験を劇的改善 |

**根拠**: Nir Eyal「Hooked」の投資フェーズ理論。初期に価値を体験させることで、サンクコスト＋保有効果で継続率が向上。Duolingoの初期XPブーストは7日継続率を23%向上させた（2023年公開データ）。

**実装観点**: 2-AはSupabaseのRPC/Edge Function、またはクライアント側で初回判定→wallet更新。2-Bは既存の承認フローに条件分岐追加。2-CはLevelUpModalに`type: "badge"`プロップを追加して汎用化。

### 挫折ポイント3: 継続モチベーション（積み重ねの見える化不足）

| 案 | 具体案 | コンポーネント/マイクロインタラクション | 文言例 | 優先度 | ROI |
|---|-------|--------------------------------------|--------|-------|-----|
| 3-A | **週次サマリーカード** | 日曜日にChildDashboard上部に自動表示。今週のクエスト数・獲得額・連続日数をカード形式で | 「こんしゅうの まとめ: 5クエスト クリア！ 150えん かせいだよ！」 | **High** | 成長の可視化。保護者にも共有可能 |
| 3-B | **連続ログインカレンダー** | 月間カレンダーUI。クエスト完了日にスタンプ。7日連続で虹色スタンプ | 「🗓️ 7にちれんぞく！ すごいぞ！」 | **Mid** | ストリーク効果（Snapchat等で実証済み）。ただし強迫的にならない設計が重要 |
| 3-C | **キャラの成長ダイアリー** | キャラクターの見た目変化を時系列で振り返れるギャラリー。「Lv1のころ」「Lv2になったひ」のスナップショット | 「📖 ぼうけんにっき — きみの せいちょうの きろく」 | **Low** | 長期的な愛着形成。実装コストは中〜高 |

**根拠**: BJ Fogg「Tiny Habits」— 小さな成功を祝うことで習慣ループが強化される。ストリーク機能はDuolingoで継続率を34%向上（ただし「ストリーク不安」の副作用あり→休息日の設計が必要）。

**実装観点**: 3-Aは`otetsudai_task_logs`から週次集計、新コンポーネント追加。3-Bはカレンダーコンポーネント新規作成（外部ライブラリ不要、View+Flexboxで実装可能）。3-Cはスクリーンショット保存が必要でコスト高。

### 挫折ポイント4: 株の抽象性（配当・複利が目に見えない）

| 案 | 具体案 | コンポーネント/マイクロインタラクション | 文言例 | 優先度 | ROI |
|---|-------|--------------------------------------|--------|-------|-----|
| 4-A | **「ふやすの木」メタファー** | `invest_balance`を「木の成長」で表現。0円=種、100円=芽、500円=若木、1000円=大木、配当時に実が落ちるアニメーション。SVGで実装（CharacterSvg類似） | 「🌳 きみの きが そだって じつが なったよ！ +5えん！」 | **High** | 抽象概念の具象化。子供が「育てる」感覚を得られる。SDT有能感に直結 |
| 4-B | **配当アニメーション付きグラフ** | WalletDetailに折れ線グラフ追加。`react-native-svg`（既存依存）で実装。配当発生時にグラフが跳ねるアニメーション。元本（青）と利益（緑）を色分け | 「📈 ふやすグラフ — おかねが じぶんで ふえてるよ！」 | **Mid** | 視覚的に複利効果を実感。ただし低年齢児にはグラフの理解が困難 |
| 4-C | **キャラ連動型配当通知** | 配当発生時にキャラクターが専用セリフで通知。「おかねの きが じつを つけたよ！」＋キャラの喜びポーズ。ChildDashboardのmoodと連動 | 「[勇者名]『やったー！ きみの おかねが 5えん ふえたよ！』」 | **High** | 既存キャラシステムとの統合でコスト低。擬人化による親しみやすさ |

**根拠**: Piaget認知発達理論 — 具体的操作期（7-11歳）の子どもは抽象概念を具体物に置き換えることで理解する。「木の成長」メタファーはGoHenry（英国子供向けフィンテック）の「Money Tree」機能で効果実証済み。

**実装観点**: 4-Aは新SVGコンポーネント（CharacterSvg同様のアプローチ）。成長段階は5段階で十分。4-Bは`react-native-svg`のPath+Animatedで折れ線グラフ描画。外部ライブラリ不要。4-Cは既存のキャラクターシステム＋stampNotifs類似の通知カードで低コスト実装可能。

## 4. ゲーミフィケーション × エージェンシー設計

### 自己決定理論（SDT）に基づく動機づけ移行モデル

```
Phase 1: 外発的動機              Phase 2: 取り入れ的動機       Phase 3: 内発的動機
(報酬のため)                     (「やるべき」認識)            (「やりたい」自発性)
┌──────────────────┐             ┌──────────────────┐          ┌──────────────────┐
│ ・クエスト報酬     │  ────────► │ ・習慣化          │ ──────►  │ ・自分でクエスト   │
│ ・レベルアップ     │    4-6週   │ ・連続ボーナス     │  2-3ヶ月  │   提案             │
│ ・バッジ収集      │            │ ・親の承認         │          │ ・報酬額の交渉     │
│ ・特別クエスト     │            │ ・キャラ育成       │          │ ・貯金目標設定     │
└──────────────────┘             └──────────────────┘          │ ・投資判断         │
                                                              └──────────────────┘
```

### SDT 3要素別の設計提案

#### A. 自律性（Autonomy）— 「じぶんで きめられる」

| 施策 | 具体的UI/UX | 対象Phase | 優先度 | ROI |
|-----|-----------|----------|-------|-----|
| **クエスト選択の自由** | 複数クエストから「きょうやるクエスト」を自分で選ぶUI。チェックボックスではなく、スワイプで「やる！」に入れるUI | Phase 1-2 | **High** | 自己決定感の醸成。Fortniteの「デイリーチャレンジ選択」と同パターン |
| **ねあげリクエスト（既存）の強化** | 現在のPriceRequestModalに「りゆう」入力欄を追加。交渉スキルの育成 | Phase 2-3 | **Mid** | 既存機能の拡張。金融リテラシー教育との連動 |
| **クエスト提案機能** | 子どもが新しいクエストを親に提案できるフォーム。「こんな おてつだいは どう？」 | Phase 3 | **Mid** | エージェンシーの最終形。親の承認フローと統合 |
| **貯金目標の自己設定（既存）** | SavingGoalModalの活用促進。ダッシュボードに「ゆめのもくひょう」として常時表示 | Phase 2-3 | **High** | 長期目標の自己決定。「ためる」の意味づけ |

#### B. 有能感（Competence）— 「できた！つよくなった！」

| 施策 | 具体的UI/UX | 対象Phase | 優先度 | ROI |
|-----|-----------|----------|-------|-----|
| **スキルツリー表示** | バッジ5種を線で繋いだツリーUI。未獲得は暗転＋条件表示。「つぎの そうび」が見える | Phase 1-2 | **High** | 「あとちょっと！」効果。ゴール勾配効果（Goal Gradient Effect） |
| **マイクロ祝福の強化** | クエストクリア時のバナーを全画面短時間（1.5秒）のセレブレーション。キャラが喜ぶポーズ＋コイン演出 | Phase 1 | **High** | BJ Fogg「Tiny Habits」の「Celebration」。低コスト高効果 |
| **週次成長レポート** | 「こんしゅうのきみ」カード。クエスト数、獲得額、レベル進捗、前週比較 | Phase 2-3 | **Mid** | メタ認知の促進。親への共有で承認欲求も充足 |
| **難易度自動調整** | クエストクリア率に基づき、特別クエストの難易度を推奨。「★★にちょうせんしてみる？」 | Phase 2-3 | **Low** | フロー理論（Csikszentmihalyi）。実装コスト高 |

#### C. 関係性（Relatedness）— 「みんなと つながってる」

| 施策 | 具体的UI/UX | 対象Phase | 優先度 | ROI |
|-----|-----------|----------|-------|-----|
| **親のスタンプ演出強化（既存改善）** | 現在のstampNotifsをカード形式から「吹き出し」形式に。キャラが親のメッセージを代読 | Phase 1 | **High** | 既存機能の演出強化。親子の感情的つながり |
| **きょうだいランキング（オプション）** | 家族内の週間クエスト数ランキング。順位ではなく「みんな がんばったね！」の協力型表示 | Phase 2-3 | **Mid** | 競争ではなく協力。チームとしての家族意識 |
| **家族ミッション** | 家族全員で達成する大型クエスト。「おうちを みんなで そうじ！」達成で全員にボーナス | Phase 3 | **Mid** | 共同達成感。帰属意識の最上位 |
| **子どもの返信（既存）の親側表示強化** | 親ダッシュボードに子どもの反応を目立つカードで表示。「こどもの こえ」セクション | Phase 1-2 | **High** | 既存データの表示最適化のみ。低コスト高効果 |

**根拠**: Deci & Ryan (2000) Self-Determination Theory。外発→内発の動機づけ移行は、3要素が同時に満たされたときに加速する。教育アプリでのSDT適用はClassDojo、Duolingoで効果実証済み。特に子ども向けでは「関係性」（親の承認）が最も強い継続因子（Przybylski et al., 2010）。

## 6. KPI / 北極星指標

### 北極星指標（NSM: North Star Metric）

> **「週3回以上クエストを自走完了する子どもの割合」**（Weekly Active Questers率）

**選定理由**: この指標が改善されれば、以下が同時に改善される：
- 継続率（アクティブ = 継続中）
- エージェンシー（「自走」= 親の介入なし）
- コアループ稼働（クエスト完了 = 報酬 → 成長の前提）

### 階層化KPIフレームワーク

#### Tier 1: CX健全性（週次モニタリング）

| 指標 | 定義 | 目標値 | 計測方法 |
|-----|------|-------|---------|
| **WAQ率**（Weekly Active Questers） | 週3回以上クエスト完了した子供 / 全登録子供 | 60%以上 | `task_logs` WHERE status='approved' GROUP BY week |
| **D7継続率** | 初回クエスト完了から7日後もアクティブ | 70%以上 | `task_logs` の初回日+7日のアクティビティ |
| **D30継続率** | 同上30日後 | 50%以上 | 同上+30日 |
| **親フィードバック率** | 承認時にスタンプ/メッセージを付けた割合 | 80%以上 | `task_logs` WHERE approval_stamp IS NOT NULL / total approved |

#### Tier 2: エージェンシー指標（月次モニタリング）

| 指標 | 定義 | 目標値 | 計測方法 |
|-----|------|-------|---------|
| **自走度スコア** | 親の介入なしでクエスト→完了した割合 | 月を追うごとに上昇 | ログイン→クエスト完了の間に親操作がないセッション |
| **ねあげリクエスト発生率** | 子どもが報酬交渉を開始した割合 | 10%以上 | `tasks` WHERE proposal_status='pending' |
| **貯金目標設定率** | 子どもが自分で貯金目標を設定した割合 | 30%以上 | `saving_goals` の登録数 |
| **つかうリクエスト発生率** | 自発的な支出申請の割合 | 20%以上 | `spend_requests` の発生頻度 |

#### Tier 3: 多様性包摂度（四半期モニタリング）

| 指標 | 定義 | 目標値 | 計測方法 |
|-----|------|-------|---------|
| **特性別継続率格差** | 各特性グループの継続率の最大差 | 15ポイント以内 | ユーザー属性タグ × 継続率のクロス集計 |
| **アクセシビリティ操作完了率** | VoiceOver/TalkBack使用時のタスク完了率 | 90%以上 | A11yテストセッションでの計測 |
| **操作エラー率** | 意図しないタップ/誤操作の発生率 | 5%以下 | クライアントログ（undo/back操作の頻度） |
| **NPS（保護者）** | 保護者のNet Promoter Score | 50以上 | 四半期アンケート |

#### Tier 4: フィンテック教育効果（半期モニタリング）

| 指標 | 定義 | 目標値 | 計測方法 |
|-----|------|-------|---------|
| **貯金達成率** | 設定した貯金目標を達成した割合 | 40%以上 | `saving_goals` WHERE is_achieved=true |
| **3ウォレット活用率** | つかう/ためる/ふやす全てに残高がある割合 | 50%以上 | `wallets` WHERE all 3 balances > 0 |
| **投資理解度** | 「ふやす」を1回以上利用した割合 | 30%以上 | `transactions` WHERE type='invest' |

## React Native 実装メモ

| 技術要素 | 実装方針 |
|---------|---------|
| **Animated** | `Animated.spring` for button scale, `Animated.timing` for celebrations. Reanimated不要（現時点では） |
| **expo-haptics** | `npx expo install expo-haptics`. `Haptics.impactAsync(Medium)` for taps, `notificationAsync(Success)` for completions |
| **accessibilityLabel** | 既存のText/TouchableOpacityに属性追加。新コンポーネント不要 |
| **AsyncStorage** | 既存活用。`onboarding_step`, `a11y_settings` キー追加 |
| **SVG（ふやすの木）** | 既存の`react-native-svg`で実装。CharacterSvgと同アーキテクチャ |
| **グラフ** | `react-native-svg`のPath/Circle/Lineで自前実装。外部チャートライブラリ不要 |
| **prefers-reduced-motion** | `AccessibilityInfo.isReduceMotionEnabled()` で判定→Context共有 |
| **フォーカス管理** | `useRef` + `AccessibilityInfo.setAccessibilityFocus()` |
