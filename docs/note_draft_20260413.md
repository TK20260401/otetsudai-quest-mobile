# 【開発記録】おこづかいクエスト モバイル版 — 実機テスト&レスポンシブ対応の1日

## はじめに

「おこづかいクエスト」は、子どものお手伝いをRPG風のクエストに見立てた、親子で使うマネー教育アプリです。Web版（Next.js）に続き、React Native + Expo でモバイル版を開発中。

今日は **Expo Go での実機テスト環境構築** から始まり、**縦画面レスポンシブ対応**、**特別クエスト設定機能**、**UI漢字化+ルビ対応** まで一気に進めました。

---

## 1. Expo Go で実機テスト環境を構築

### ハマりポイントの連続

開発PCで `npx expo start` を叩くだけ...のはずが、いくつもの壁にぶつかりました。

**Node.js v24 問題**
Expo SDK 54 は Node.js v24 に対応しておらず、バンドルが0%のまま固まる現象が発生。Node v22 に切り替えることで解決。

**ネットワーク接続**
- `exp://127.0.0.1:8081` — ローカルホストなのでスマホから接続不可
- `--tunnel` モード — ngrok 経由だがタイムアウト
- `--lan` モード — 同一Wi-Fiで解決！

実機でアプリが動いた瞬間の感動は格別でした。

---

## 2. 縦画面でのレスポンシブ対応

横画面（ランドスケープ）では問題なかったUIが、縦画面（ポートレート）では崩壊。

### Before
- 「おこづかいクエスト」のタイトルが「おこづかいク / エスト」と不自然に改行
- クエスト作成モーダルのタイトルが2行に折れる
- モーダル下部のコンテンツが画面外に切れてスクロール不可
- ログイン画面の「おやこうざ」テキストが枠からはみ出し

### 対策

**レスポンシブフォントユーティリティ `rf()` を導入**

```typescript
const BASE_WIDTH = 375; // iPhone SE基準
export const rf = (size: number): number => {
  const scale = SCREEN_WIDTH / BASE_WIDTH;
  return Math.round(size * Math.min(scale, 1.3));
};
```

加えて、React Native の `adjustsFontSizeToFit` + `numberOfLines` を活用し、テキストが自動的にコンテナに収まるようにしました。

## 3. 親画面の漢字化

子ども画面はひらがな中心ですが、親画面は大人が使うので漢字に統一しました。

| Before | After |
|--------|-------|
| しょうにん | 承認 |
| ほうしゅう（えん） | 報酬（円） |
| くりかえし | 繰り返し |
| なんいど | 難易度 |
| きかん（かいし） | 期間（開始） |
| ねあげリクエスト | 値上げリクエスト |

全40箇所以上を一つずつ変換。地味だけど、大人が使う画面としてのUXが大幅に向上しました。

## 4. カレンダーによる日付選択

特別クエストの開始日・終了日を、テキスト入力からカレンダーピッカーに変更。

`@react-native-community/datetimepicker` を導入し、iOS ではインラインカレンダー、Android ではネイティブのデートピッカーが表示されます。

日付のクリアボタン（✕）も追加し、「期限なし」の設定も簡単に。

---

## 5. 特別クエスト設定パネル

親が難易度ごとに特別クエストの表示/非表示を制御できる設定パネルを実装。

```
┌─────────────────────────────────────────┐
│  ★ 特別クエスト設定                      │
│                                         │
│  特別クエスト全体   [====ON====]         │
│  ★  簡単           [====ON====]         │
│  ★★ 普通           [====ON====]         │
│  ★★★ 難しい        [===OFF====]         │
└─────────────────────────────────────────┘
```

**仕組み:**
- Supabase に `otetsudai_family_settings` テーブルを新規作成
- 全体OFFで子ども画面から特別クエストが完全非表示
- 難易度別に個別制御可能
- 親画面ではOFFの難易度もグレーアウトで表示（管理のため）

---

## 6. 子ども画面のルビ（ふりがな）対応

教育アプリとして、子どもが漢字を学べるようにルビを実装。

React Native にはHTMLの `<ruby>` タグがないため、カスタムコンポーネントを自作：

```tsx
<RubyText
  style={styles.walletLabel}
  parts={[["貯", "た"], "める"]}
/>
```

漢字の上に小さくふりがなが表示されます。「財布（さいふ）」「装備（そうび）」「特別（とくべつ）」など、主要な漢字にルビを追加しました。

---

## 7. RLS ポリシー問題の解決

クエストを作成しても画面に表示されない不具合が発生。エラーハンドリングを追加したところ、原因は **Supabase の Row Level Security (RLS)** でした。

Web版は Supabase Auth でログインするため `get_my_family_id()` が機能しますが、モバイル版はPINログインで Auth を使っていないため、INSERT が拒否されていました。

RLS ポリシーを `WITH CHECK (true)` に変更して解決。
