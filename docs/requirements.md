# おこづかいクエスト Mobile 要件定義書（次期版 / To-be）

> 本書は **モバイル版（otetsudai-quest-mobile）** の要件定義。プロダクト共通の機能/非機能/技術要件の **正本は Web 版** `otetsudai-bank/docs/requirements.md`。本書は「共通コア要件の参照」＋「モバイル固有要件」を定義する（dual update 方針）。
> リリース方針: **速やかに公開**。TestFlight 配布中（Phase 3）の延長線で、ストア公開ブロッカーの解消を最優先とする。
> 最終更新: 2026-06-19

---

## 0. 方針

- 共通の差別化3軸（①特別支援ファースト ②カード非発行＝規制フリー ③日本語ネイティブのクエスト世界観）と機能/非機能要件は Web 版正本に準拠する。
- モバイル版は **ネイティブならではの価値（プッシュ通知・常時携帯・オフライン・ストア配布）** を上乗せして「速やかに公開」する。
- 共通コアの変更は Web 版とセットで反映する（feedback: 両版同時更新）。

## 2. 非機能要件（モバイル固有）

- **NFR-M01 法務**: Web 版 NFR-L を踏襲（換金/送金/対外決済なし＝資金移動業・前払式の登録回避）。加えて **App Store / Google Play の子供向けカテゴリ規約**（広告・外部リンク・課金・データ収集の制約）を順守。
- **NFR-M02 アクセシビリティ**: 全漢字ルビ、最小タップ領域、OS のフォントサイズ/読み上げ（VoiceOver / TalkBack）連携、`reduce motion` 連携。
- **NFR-M03 セキュリティ**: Supabase RLS 共用、トークン安全保管（SecureStore 等）、anon_key をクライアント限定・private 操作はサーバ経由。
- **NFR-M04 性能**: 起動・画面遷移の体感速度、低スペック端末・古い OS への配慮。
- **NFR-M05 配布運用**: EAS ビルド/サブミット、OTA 更新の運用、クラッシュ監視。

## 3. 技術要件（モバイル固有）

- フレームワーク＝Expo（SDK 54 系）/ React Native / Expo Router / TypeScript。
- バックエンド＝Supabase（Web 版と共用、RLS）。
- ビルド/配布＝EAS（eas.json）、iOS/Android ネイティブプロジェクト同梱。
- 通知＝Expo Notifications（プッシュ）。
- 課金＝アプリ内課金（ストア準拠）。
- AI＝Claude API（サーバ経由）。
- 共通描画＝react-native-svg、効果音＝expo-av、保存＝AsyncStorage。

### 外部連携
- **TR-M-E01**: Supabase（共用）/ Edge Function。
- **TR-M-E02**: Expo Push / ストア課金 / Claude API。
