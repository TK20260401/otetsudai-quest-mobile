## 実装プロンプト

### プロンプト① デザイン・UI改善（親ページ＋ログイン画面）

```
おこづかいクエスト（otetsudai-bank）のTestFlightフィードバックに基づき、以下のUI改善を実施してください。

### 親ページ（app/parent/tasks/page.tsx 周辺）

1. **ふりがな削除（親ページ全体）**
   - 親ページ（app/parent/page.tsx、app/parent/tasks/page.tsx）では ruby-text によるふりがな表示を除去
   - 親は大人なのでふりがなは不要。子供ページのみふりがな維持

2. **「期限なしなら なしでOK」の文言位置修正**
   - クエスト作成・編集フォームで、この補足テキストを「期間（始まり／終わり）」の入力欄の横（すぐ近く）に移動

3. **カレンダータブ削除 & アイコン配置改善**
   - クエスト編集画面の「カレンダー」タブを削除
   - 代わりに、期間入力欄の左横にカレンダーアイコンを配置
   - 他のカラム（タイトル、報酬、繰り返し等）にも各項目に合ったアイコンを左横に揃えて配置し、統一感を出す

4. **「誰に」フィールドをプルダウン化**
   - 新しいクエスト作成時の「誰に（対象の子供）」選択UIを、現在の方式からプルダウン（Select / ドロップダウン）に変更

5. **「＄」アイコンの見直し**
   - クエストの報酬表示で使っている「＄」アイコンを削除、または「円」やコイン風のゲームらしいアイコンに置き換え

6. **アイコン全体をゲームらしいデザインに**
   - 各クエストや機能のアイコンを、Habitica風のゲームらしいテイストに変更（lib/task-icons.ts 等）
   - Lucide Reactの中からゲーム感のあるアイコンを選定するか、適切な絵文字に変更

7. **報酬変更の導線改善**
   - 報酬金額の変更を「設定」画面経由ではなく、親ダッシュボードや子供の残高表示エリアから直接変更できるUIにする
   - お金の表示部分にタップ可能な編集アイコンを追加し、インライン編集またはモーダル編集を可能にする

### ログイン画面（app/login/page.tsx）

8. **親と子供のログイン経路を分離**
   - ログイン画面で最初に「おやモード」「こどもモード」を選択するUIを追加
   - 親はメール/パスワード入力、子供はファミリー選択→PIN入力 のように導線を分ける

9. **ログインボタンの文言変更**
   - 「ログイン」を「クエストをはじめる！」のようなゲーム風の文言に変更

10. **使う/貯める/増やすの説明文の改行修正**
    - ログイン画面またはサインアップ完了画面にある「使う」「貯める」「増やす」の説明テキストの不自然な改行を修正
    - テキストが自然に流れるようにレイアウト調整

各変更後、README.mdのバージョン履歴を更新し、git pushまで実施してください。
```

### プロンプト② 動作バグ修正

```
おこづかいクエスト（otetsudai-bank）のTestFlightフィードバックで報告された動作バグを修正してください。

### 親ページのバグ（app/parent/tasks/page.tsx 周辺）

1. **カレンダー表示崩れ**
   - クエストの期間で開始日を選択する際、カレンダーUIの表示が崩れる問題を修正
   - DatePickerまたはカレンダーコンポーネントのスタイリング/レイアウトを確認・修正

2. **「期限なし」の選択解除不可**
   - 一度「期限なし」に設定すると、その選択を解除して期限付きに戻せない問題を修正
   - トグルまたはクリアボタンで解除できるようにする

### 子供画面のバグ（app/child/[childId]/page.tsx、components/saving-goal.tsx 周辺）

3. **財布の目標作成：タイトル未入力バリデーション**
   - 貯金目標（SavingGoal）作成時、タイトルが未入力のまま保存しようとした場合にバリデーションエラー（注意マーク等）が表示されるよう修正

4. **財布の目標作成画面のエラー**
   - 目標作成画面でエラーが頻発する問題を調査・修正
   - エラーハンドリング、nullチェック、APIレスポンスの処理を確認

5. **クエストクリア時の「次のレベルまで」UI未反映**
   - クエスト完了承認後、経験値バーや「次のレベルまで○○ポイント」の表示がリアルタイムで更新されない問題を修正
   - components/level-display.tsx や親の承認処理後のデータ再取得を確認
   - 親とのやりとり部分（承認ダイアログ等）でも同様にレベル情報が反映されるよう対応

6. **「使う」ボタン押下後の画面フリーズ**
   - 子供画面で「使う」ボタンクリック後、戻ると画面が固まる問題を修正
   - SpendRequestダイアログのクローズ処理、stateの不整合、非同期処理の競合を調査

各修正後、README.mdのバージョン履歴を更新し、git pushまで実施してください。
```
