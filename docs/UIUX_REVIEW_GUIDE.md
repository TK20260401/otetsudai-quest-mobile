# おこづかいクエスト モバイル版 — UI/UX/CX レビューガイド

## レビューの目的

実機で触ってもらい、以下の観点でフィードバックを集める：
1. **離脱ポイント** — 迷う・詰まる・やめたくなる箇所はないか
2. **直感性** — 説明なしで操作できるか
3. **ユニバーサルデザイン（UD）** — 子ども（未就学児〜小6）が使えるか

---

## 対象ユーザー

| ユーザー | 年齢 | 操作レベル |
|---------|------|-----------|
| 子ども | 未就学児〜小6 | タップ・スワイプのみ |
| 親 | 20代〜40代 | スマホ操作に慣れている |

---

## レビューシナリオ

### シナリオ1: 子どもの初回体験（3分以内に完了できるか）

1. アプリを開く → ランディング画面が表示される
2. 「ログイン」をタップ
3. 家族を選ぶ → 自分の名前をタップ
4. ダッシュボードが表示される
5. クエスト一覧を見て「クリア！」ボタンをタップ

**チェックポイント:**
- [ ] ログインまで迷わずたどり着けたか
- [ ] ボタンの文字が読めたか（ルビは見えるか）
- [ ] 「クリア！」ボタンがすぐ見つかったか
- [ ] 画面遷移に違和感はなかったか
- [ ] キャラクターやレベル表示に興味を持ったか

### シナリオ2: 親のクエスト作成（5分以内に完了できるか）

1. 親アカウントでログイン
2. 「クエスト」タブをタップ
3. 「+ クエスト」ボタンをタップ
4. クエスト名・報酬を入力して「作成」
5. 一覧に反映されることを確認

**チェックポイント:**
- [ ] タブの意味（承認/クエスト/子ども）がすぐわかったか
- [ ] クエスト作成フォームで迷う項目はなかったか
- [ ] 「繰り返し」「誰に？」の選択は直感的だったか
- [ ] 特別クエストの作り方がわかったか
- [ ] 作成後、一覧にすぐ反映されたか

### シナリオ3: 親の承認フロー

1. 子どもがクエストを「クリア！」した後
2. 親画面の「承認」タブに通知が表示される
3. スタンプを選んで「承認」をタップ

**チェックポイント:**
- [ ] 承認待ちがあることにすぐ気づけたか
- [ ] スタンプの選択は楽しかったか
- [ ] 承認後、子どもの画面に反映されたか

### シナリオ4: 特別クエスト設定

1. 親画面 →「クエスト」タブ
2. 「★ 特別クエスト設定」パネルを確認
3. トグルをON/OFF
4. 子ども画面で反映を確認

**チェックポイント:**
- [ ] 設定パネルの存在に気づいたか
- [ ] 全体OFF → 子ども画面で非表示になったか
- [ ] 難易度別の制御が理解できたか
